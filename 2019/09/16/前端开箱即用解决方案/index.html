前端开箱即用解决方案

发表于 2019-09-16

特性

  • 最佳实践：良好的工程实践助你持续产出高质量代码
  • Mock 数据：实用的本地数据调试方案（支持mockjs随机数据语法）
  • UI 测试：自动化测试保障前端产品质量（jest + puppeteer）
  • 代码规范校验：收敛lint规则配置到vbuilder统一管理，全站统一校验规则eslint / stylelint

模板

使用

.prettierrc为项目中自定义样式配置，v lint执行时自动读取vbuilder中统一lint规则校验代码

v mock新增功能，可解析mockjs语法生成随机内容（应用于各种边界条件测试），访问对应mock资源后会在相同目录下生成*-result.json作为访问结果文件（用于追溯上一次请求的数据结果）

Mock Server读取项目根目录配置（.mockrc） 增加是否代理到其他环境选项（proxy开启后 server读取项目中mock文件，读取不到的情况下转发对应host接口结果返回）
新增mock-add [path] 命令 快速创建mock文件

gitignore新增如上配置，lint-result为v lint校验后生成的报表文件，*-result.json为模拟数据结果文件

如何在项目中应用

  • 新项目直接v init 初始化项目即可使用
  • 老项目可应用新升级后的 v mock / v lint 命令
